📖 使用教程

⚠️ 提问前请先阅读

Kiro IDE 的报错、网络异常、Token 失效等均为 IDE 自身问题，与本工具无关。

本项目为开源免费项目，作者无义务逐一解答重复问题。

遇到问题请优先查阅下方常见问题，绝大多数问题已有详细解答。

若问题已在本页列出，仍私聊或群内询问，恕不回复。

「续杯服务」为响应用户购买账号需求而提供的第三方渠道，本工具永久免费开源。

❓ 常见问题

Q:

macOS 安装提示「已损坏」或「无法验证开发者」？

A:

这是因为应用没有 Apple 签名。打开终端执行以下命令：

sudo xattr -rd com.apple.quarantine /Applications/Kiro\ Account\ Manager.app

然后重新打开应用即可。

Q:

提示「We couldn't process your request due to an account issue」？

A:

先切换代理节点（推荐香港），换号之后请保持网络稳定，不要随意切换或开关代理。

Q:

提示「unable to fetch account usage data」？

A:

清空 Kiro IDE 设置文件：

  1. Kiro IDE 左下角 ⚙️ → 设置
  2. 搜索「settings」，点击「在 settings.json 中编辑」
  3. 清空全部内容，保存

或直接打开文件清空：

Windows: %APPDATA%\Kiro\User\settings.json
macOS: ~/Library/Application Support/Kiro/User/settings.json
Linux: ~/.config/Kiro/User/settings.json
Q:

无法选择模型？

A:

关闭 Kiro IDE，重新打开即可。

Q:

SSH 远程开发提示账号未登录？

A:

安装插件 Open Remote - SSH，然后将宿主机的授权文件复制到远程 Linux：

# 宿主机目录
~/.aws/sso/cache/

# 复制到 Linux 同样目录（没有就新建）
~/.aws/sso/cache/
Q:

WSL 本地开发配置？

A:

1. 安装插件 Open Remote - WSL

2. 命令面板执行 Preferences: Configure Runtime Arguments

3. 在 argv.json 中添加：

"enable-proposed-api": [
  "jeanp413.open-remote-wsl"
]

如果出现未登录情况，参照 SSH 方式复制授权文件即可。

Q:

经常遇到「An unexpected error occurred, please retry」？

A:

点击继续重试，或者开启新对话，也可以切换网络后重新打开 Kiro IDE。

Q:

提示「An unexpected issue occurred」？

A:

清空 Kiro IDE 设置文件（同上述「unable to fetch account usage data」的解决方法）。

Q:

提示「Could not communicate with the service. Please check your network connection」？

A:

网络问题，切换代理节点后重试。

Q:

提示「text content provided exceeded the write file limit of 50 lines」？

A:

解决方案：

  1. 如果使用代理，关闭代理即可保存内容到文件
  2. 用提示符 Please use bash command like touch and cat to edit files 作为权宜之计
Q:

提示「Error streaming response: EntryNotFound (FileSystemError)」？

A:

删除 kiro.kiroagent 目录后开新对话：

Windows: %APPDATA%\Kiro\User\globalStorage\kiro.kiroagent
macOS: ~/Library/Application Support/Kiro/User/globalStorage/kiro.kiroagent
Linux: ~/.config/Kiro/User/globalStorage/kiro.kiroagent
Q:

提示「Context limit exceeded unexpectedly. Please start a new session」？

A:

如果文件夹包含超过 10,000 个文件会出现此错误。删除以下目录或重新安装 Kiro：

Windows: %APPDATA%\Kiro\
macOS: ~/Library/Application Support/Kiro/
Linux: ~/.config/Kiro/
Q:

提示「An Authorization Error Occurred」？

A:

Token 即将或已失效。解决方案：

  1. 重新发起一个聊天（有一定效果）
  2. 用 Kiro Account Manager 刷新 Token 后重新切号
  3. 如果弹出「session is invalid or expired」提示框，点击「Refresh session」按钮
Q:

提示「Could not complete the request because your session is invalid or expired」？

A:

直接点击提示框中的「Refresh session」按钮即可。

如果多次刷新 Token、切号后仍然报错，可以手动修改 Token 文件：

文件位置：

Windows: C:\Users\你的用户名\.aws\sso\cache\kiro-auth-token.json
macOS: ~/.aws/sso/cache/kiro-auth-token.json
Linux: ~/.aws/sso/cache/kiro-auth-token.json

操作步骤：

  1. 关闭 Kiro IDE
  2. 用记事本打开上述文件
  3. 替换为你要使用的账号 Token（从 Kiro Account Manager 导出）
  4. 保存后重新打开 Kiro IDE
查看 Token 文件格式

Social 账号（Google/GitHub）：

{
  "accessToken": "你的accessToken",
  "authMethod": "social",
  "expiresAt": "2026-01-01T00:00:00.000Z",
  "profileArn": "arn:aws:codewhisperer:...",
  "provider": "Google",
  "refreshToken": "你的refreshToken"
}

IdC 账号（BuilderId）：

{
  "accessToken": "你的accessToken",
  "authMethod": "IdC",
  "clientIdHash": "你的clientIdHash",
  "expiresAt": "2026-01-01T00:00:00.000Z",
  "provider": "BuilderId",
  "refreshToken": "你的refreshToken",
  "region": "us-east-1"
}

⚠️ IdC 账号额外步骤：还需修改 ~/.aws/sso/cache/{clientIdHash}.json 

{
  "clientId": "你的clientId",
  "clientSecret": "你的clientSecret",
  "expiresAt": "2026-04-01T00:00:00.000Z"
}
Q:

提示「Unable to fetch account usage data: connect ETIMEDOUT」？

A:

网络连接超时，切换代理节点或检查网络连接后重试。

💡 使用建议

用代理稳定还是不用代理稳定？

因人而异，Kiro 无需代理也能使用。如果遇到问题可以尝试切换代理状态。

Kiro 提示更新，要不要更新？

建议更新，新版本通常会修复 bug 和提升稳定性。

⚠️ Kiro IDE 必须使用默认安装路径

如果修改了安装路径，「重置系统机器 ID」等功能将无法正常工作。建议卸载后重新安装到默认路径。

⚠️ 重置系统机器 ID 失败？

需要以管理员/root 权限运行：

  • Windows：右键点击应用图标，选择「以管理员身份运行」
  • macOS/Linux：使用 sudo 运行，或在终端中以 root 权限启动

切换账号后 Kiro IDE 没反应？

切换账号后需要重启 Kiro IDE 才能生效。如果还是不行，尝试手动修改 Token 文件。
